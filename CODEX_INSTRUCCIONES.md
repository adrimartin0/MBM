# Instrucción maestra para Codex

Quiero que mejores esta web de muestra manteniendo el proyecto simple y ordenado.

## Contexto

Es una web estática creada con HTML, CSS y JavaScript. La quiero usar como práctica para aprender flujo real con GitHub, Codex y GitHub Pages.

## Objetivo

Convertir esta plantilla en una landing page más profesional, moderna y responsive.

## Requisitos obligatorios

- Mantén la estructura simple:
  - `index.html`
  - `styles.css`
  - `script.js`
  - `README.md`
- No uses frameworks.
- No añadas dependencias externas.
- No rompas GitHub Pages.
- Mantén el archivo `index.html` en la raíz del repositorio.
- El diseño debe verse bien en móvil y ordenador.
- El código debe estar limpio, ordenado y sin texto de relleno raro.
- No elimines contenido útil sin justificarlo.
- Revisa el resultado antes de terminar.

## Mejoras deseadas

- Mejorar el diseño visual general.
- Añadir una sección de “proyectos” o “servicios”.
- Añadir una sección de preguntas frecuentes.
- Mejorar el footer.
- Añadir pequeñas interacciones con JavaScript si tienen sentido.
- Mejorar SEO básico:
  - `title`
  - `description`
  - estructura de encabezados
  - textos claros
- Mejorar accesibilidad:
  - botones claros
  - navegación usable
  - buen contraste
  - etiquetas correctas

## Entrega esperada

Cuando termines:
- Explica qué archivos has cambiado.
- Explica qué mejoras has hecho.
- Indica si hay algo pendiente.
- Confirma que la web sigue siendo publicable con GitHub Pages.
